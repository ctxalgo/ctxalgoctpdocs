static rebalance(account_positions, current_strategy_positions, expected_positions)¶

The position rebalance is defined as:

Let P(s, i, d) be the re-balanced position for instrument i of strategy s, on d, where i indexes instruments, such as cu1601, cu1602, s indexes the list of strategies, and d indexes historical or today. In other words, s in set of strategies {s1, s2, s3}, i in set of possible instruments in the account. d in set of {historical, today}.

Let A(i, d) be the position from the account. Data is from account_positions.

Let C(s, i, d) be the current positions for instrument i in strategy s on day d. Data is from current_strategy_positions.

Let E(s, i) be the expected positions for instrument i in strategy s. Note that expected positions does not distinguish between historical or today. Data is from expected_positions.

The position rebalance is a minimization problem which tries to find an assignment for P(s, i, d) to minimize

sum_over_s_i_d (C(s,i,d) - P(s,i,d))^2 + sum_over_s_i (E(s,i) - P(s,i,d=historical) - P(s,i,d=today))^2

close to current close to expected positions strategy positions

subject to: constraint 1: forall i . A(i, d=today) == sum_over_s (P(s, i, d=today)) constraint 2: forall i . A(i, d=yesterday) == sum_over_s (P(s, i, d=yesterday))

For example, if an account has the following positions: { ‘cu’: {‘historical_volume’: 5, ‘today’: 7},

‘rb’: {‘historical_volume’: 3, ‘today_volume’: 2} }

And there are two strategies s1 and s2, and their current positions are: s1: { ‘cu’: {‘historical_volume’: 6, ‘today_volume’: 1} }

s2: { ‘cu’: {‘historical_volume’: 4, ‘today_volume’: 2},
‘rb’: {‘historical_volume’: 4,’today_volume’: 1} }

And expected positions for s1 and s2 are: s1: {‘cu’: 3, ‘rb’: 2} s2: {‘cu’: 10, ‘rb’: 6}

Then the following is a possible assignment for position rebalance (may not necessarily the optimal): s1: { ‘cu’: {‘historical_volume’: 2, ‘today_volume’: 1},

‘rb’: {‘historical_volume’: 1, ‘today_volume’: 0} }
s2: { ‘cu’: {‘historical_volume’: 3, ‘today_volume’: 6},
‘rb’: {‘historical_volume’: 2, ‘today_volume’: 2} }

static optimize(current_historical_position, current_today_position, expected_position)¶

For certain instrument and strategy, get the best position of historical and today. Let C(instrument, strategy, d=historical)=A, C(instrument, strategy, d=today)=B, E(instrument, strategy)=C. f(x, y) = (x-A)^2 + (y-B)^2 + (x+y-C)^2 Find x and y that minimize f(x, y). x >= 0, y >= 0

case 1: expected_position is not None sqrt((x-A)^2 + (y-B)^2) is the distance between (x, y) and (A, B), sqrt((x+y-C)^2 / 2) is the distance from (x, y) to the line x+y=C So the vertical point from point (A, B) to line x+y=C is the answer. x = (C + A - B) / 2, y = (C-A+B)/2 if x < 0, y > 0, then x = 0, y=C if x > 0, y < 0, then x = C, y=0 case 2: expected_position is None x = A, y = B :param current_historical_position: int, the current historical position :param current_today_position: int, the current today position :param expected_position: the expected position :return: (int, int), (historical rebalanced position, today rebalanced position)
