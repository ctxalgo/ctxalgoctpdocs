This factory returns real CTP market and trading classes.

Initialize current. :param security_company: string, security company name. :param data_feed: AbstractDataFeed, the data feed. :param initial_capital: float, the initial capital for the trading account.

This is used to make sure the strategy report will use the current initial capital to calculate metrics.
Parameters:
  • dominant_provider – AbstractDominantProvider, the dominant provider that provides mapping from relative instrument ids, such as cu00, to actual instrument ids, such as cu1609.
  • future_info_factory – AbstractFutureInfoCalculatorFactory, the factory to create future info related object, such as volume multiples, tick sizes.

Create a remote trade executor. A remote trade executor perform trading through a remote trade execution server. Zeromq is used to do inter-process communication between the strategy and the remote trade execution server. :param name: string, the name of the trade executor. :param context: zmq context, the zmq context. :param queue_to_execution_server: string, the zmq queue address from strategy to the trade execution server.

If None, a default will be used.
Parameters:
  • queue_from_execution_server – string, the zmq queue address from execution server to strategy. If None, a default will be used.
  • proxy – bool, if True, communication is through a proxy.
Returns:

ZmqTradeExecutor, the trade executor.

Create a real factory. :param trade_executor: AbstractTradeExecutor, the trade executor. A trade executor performs trading.

  • initial_capital – float, initial capital, used to generate strategy report.
  • dominant_provider – AbstractDominantProvider, used to support trading with relative instruments, such as IF00, cu01.

Returns:

CtpRealFactory, the created factory.
