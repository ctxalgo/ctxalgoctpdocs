put_historical_ohlcs(future_id, bars, new_ohlcs_func)¶

Put ohlc bars into the ohlc cache in current object. This is only used in backtesting. In real time trading, this method is no-operation. :param future_id: string, the future id. :param bars: dict{string: dict{Periodicity or int/float: Bar}} The bars.

Outer keys are ohlc kind such as ‘time-based’ or ‘volatility-based’. Inner keys are Periodicity or volatility thresholds, inner values are the generated bars.
Parameters:new_ohlcs_func – function pointer to get ohlcs. The function pointer has two parameters: ohlc_kind and instrument_id. It returns dict{Periodicity or int/float: OHLC}, meaning dict{period or volatility threshold: OHLC}.

get_historical_ohlcs(future_id, periods, max_bar, kind, with_open_interests, dominants, profits, order_book_level)¶

Return ohlc objects with given periods. This is only used in backtesting. In real time trading, this method is no-operation. :param future_id: string, the future id. :param periods: [Periodicity], the list of periodicities under which ohlc objects are to be returned. :param max_bar: int, the maximum number of bars for each of the retrieved ohlc. :param kind: string, kind of ohlc. Valid values are ‘time-based’, ‘volatility-based’ from

OhlcGeneratorConstants.
Parameters:
  • with_open_interests – bool, if True, include open_interests field in ohlcs.
  • dominants – bool, if True, include the dominants field in ohlcs.
  • profits – bool, if True, include the profits field in ohlcs.
  • order_book_level – int, the order book level.

:return dict from Periodicity to OHLC object.
