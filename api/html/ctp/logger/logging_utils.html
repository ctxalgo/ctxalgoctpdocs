class ctxalgoctp.ctp.logger.logging_utils.LogUrlGenerator(day_separator=True, profit=False, investor_position=True, balance=True, available_money=False, pre_loaded_bars=200, max_days_per_chart=None, signal_with_price=True, stop_loss=True, chart_period=None, show_ignored_signals=True)¶

Class to generate urls for the website from strategy logs.

Initialize the url generator. :param records: [LogRecord], the list of log records read from the output.txt file. :param day_separator: boolean, if True, display vertical bar between trading days. :param profit: boolean, if True, output an auxiliary chart to display position profits. :param investor_position: boolean, if True, output an auxiliary chart to display investor positions. :param balance: boolean, if True, output an auxiliary chart to display balance and available money. :param pre_loaded_bars: int, the number of bars to load before the starting of the main data. :param max_days_per_chart: int, the maximum number of trading days to display per chart.

generate(records, sid=None, extra_indicators=None, kind='trade', lang='en')¶

Generate urls. When kind is ‘trade’, the generation also sets self.last_transactions. :param sid: string, the security id. :param extra_indicators: string, extra indicators to display.

If None, use the url_indicators information from the log.
Parameters:
  • kind – string, can be ‘signal’, ‘trade’.
  • lang – string, the language for the text in the generated url. ‘zh’ means Chinese, ‘en’ means English.
Returns:

dict from string to string, keys are url titles, values are urls.
