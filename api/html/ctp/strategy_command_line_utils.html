strategy_command_line_utils¶

class ctxalgoctp.ctp.strategy_command_line_utils.StrategyCommandLineUtils(name=True, account=True, instruments=True, positions=True, mission_control=True, logger=True, base_folder=True, data_producer=True, trade_executor=True, local_bookkeep=True, start_order_ref=True, trade_execution_server=False, ohlcs=True, dominant_provider=True, context=None, strategy_class=None, auto_parse=False, cmd_options=None)¶

Bases: object

Parameters:
  • name – bool, whether the name option is included.
  • account – bool, whether the account option is included.
  • instruments – bool, whether the instruments option is included.
  • positions – bool, whether the potions option is included.
  • mission_control – bool, whether mission control related options are included.
  • logger – bool, whether the logger option is included.
  • base_folder – bool, whether the base_folder option is included.
  • data_producer – bool, whether data producer related options are included.
  • trade_executor – bool, whether trade executor related options are included.
  • local_bookkeep – bool, whether the local_bookkeep option is included.
  • start_order_ref – bool, whether the start_order_ref option is included.
  • trade_execution_server – bool, whether options related to trade execution server are included.
  • ohlcs – bool, whether historical ohlc related options are included.
  • context – zmq Context, need to be provided if some of the command line options need zmq. If None, and a zmq context is needed, an internal context will be created. If an internal context is created, you need to call dispose_context to dispose it.
  • strategy_class – class, the class of the strategy class.
  • auto_parse – bool, if True, parse the command line options inside __init__, otherwise, you need to call self.parse() to get command line options parsed. If you need to add additional options, you need to set this to False, and after initialization, call self.add_option to add new options, and then call self.parse to parse the command line options.
  • dominant_provider – bool, if True, include options for dominant providers.
  • cmd_options – string, or [string], if provided, the values here are parsed as command line options, otherwise, use the command line options from the running program. If type is string, then it is space-separated before parsing, for example, ‘–account simnow_future –instruments cu1610’. If type is [string], then it is something like [‘–account’, ‘simnow_future’, ‘–instruments’, ‘cu1610’].
context()¶

Return a zmq Context. If a context is provided through __init__, return that context, otherwise, create an internal context. If an internal context is created. You need to call dispose_context to dispose it. :return: zmq Context.

dispose_context()¶

Dispose self._context if it is created internally.

Add an option into the parser. Usage is the same as OptionParser.add_option.

parse(cmd_options=None, ignore_argv=False)¶

Parse command line options. :param cmd_options: string or [string], if not None, use it as command line options,

otherwise use the command line of the running program.
Parameters:ignore_argv – bool, if True, do not use actual command line options sys.argv, instead, only use cmd_options. If False, use actual command line options sys.argv if there is parameters in it, otherwise, use cmd_options.
Returns:parsed options.
