optimization¶

ctxalgoctp.ctp.optimization.make_walk_forward_config(train_window, pred_window, param_specs, funs, pred_start_days=None, optimizer=None, optimizer_options=None, cons=None, regularizers=None, output_folder='training')¶

A utility function to make a dictionary about walk forward configuration. The optimizer argument can be either an optimizer object or a string that specifies the optimizer name. Specifically: “brute-force”: BruteForceOptimizer , “random-guess”: RandomGuessOptimizer, “simulated-annealing”: SimulatedAnnealingOptimizer, “hill-climbing”: HillClimbingOptimizer.

ctxalgoctp.ctp.optimization.objective_factory(start_date, end_date, last_param, data_source_factory, strategy_class, config, param_specs, funs, cons, regularizers, slippage_model=None, has_commission=True, daily_start_time=None, daily_end_time=None, message_random_delay_enabled=False)¶

Generate an objective object given start_date, end_date and previous parameter value.

This function can be used as a lambda function in the walk forward backtesting.

Parameters:
  • start_date – start date of the rolling window for the backtesting.
  • end_date – end date of the rolling window for the backtesting.
  • last_param – numpy array of the parameter value in the last rolling window.
  • data_source_factory – function, returns the data source within [start_date, end_date)
  • config, param_specs, funs, cons, regularizers (strategy_class,) – the same as in CtpBacktestingObjective.
  • daily_start_time – time, the time when the backtester simulation starts everyday. (Because walk-forward optimization uses backtester internally). If None, a default value is used.
  • daily_end_time – time, the time when the backtester simulation ends everyday. If None, a default value is used.
Returns:

a CtpBacktestingObjective object.
class ctxalgoctp.ctp.optimization.CtpBacktestingObjective(data_source, strategy_class, config, param_specs, funs, cons=None, regularizers=None, slippage_model=None, has_commission=True, daily_start_time=None, daily_end_time=None, message_random_delay_enabled=False)¶

Bases: ctxalgolib.ta.optimization.objective.ObjectiveBase

Objective function for strategy optimization in backtesting.

A function or constraint specification should be a lambda function and the argument names should be strings that are recognizable by CtpBacktestingObjective. Supported argument names are:

  1. Performance metrics in the backtesting summary report. For example, ‘transaction_count’, ‘max_drawdown’,
‘max_drawdown_percent’, ‘net_profit’, ‘average_net_profit_by_transaction’, ‘maximum drawdown duration’
  1. Parameter name in param_specs
  2. Internal representation of the parameters in the format ‘[parameter name]_internal’
  3. Others that are defined separately (we should keep these strings in a list and prepare to add more strings later): ‘maximum drawdown duration’: maximum drawdown duration in seconds.
Parameters:
  • data_source –
  • strategy_class –
  • config – strategy configuration.
  • param_specs – ParameterVector object that specifies parameters to be optimized in the config[‘parameters’] dict.
  • funs – a list of lambda functions to specify report entries (with arguments) as function output.
  • cons – a list of lambda function to specify report entries (with arguments) as constraint output.
  • regularizers – list of regularizer objective object, each for one parameter, or None if not used.
  • slippage_model – AbstractSlippageModel, the slippage model to use for the backtester. If None, VolumeBasedSlippageModel will be used.
